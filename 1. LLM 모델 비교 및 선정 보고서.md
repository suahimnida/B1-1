### <b>1. 메가존클라우드 HALO</b>

<b>개요</b>
<br>HALO는 메가존클라우드가 출시한 통합 클라우드 보안 운영 플랫폼(Security Operating System)입니다.</br>
기존 MSP가 단순히 클라우드를 관리하는 수준이었다면 HALO는 보안 진단, 보안 구축, 보안 운영, 사고 대응, 규제 대응
<br>전 과정을 하나의 플랫폼으로 제공합니다. AI 기반 오케스트레이션과 구독형 vCISO를 통해 보안 운영과 컴플라이언스를 통합 지원하는 것이 특징입니다.</br>

<b>주요 기능</b>
- CSPM(Cloud Security Posture Management)
- IAM 관리
- 취약점 진단
- IEM
- SOAR
- CNAPP
- Container Security
- Kubernetes Security
- AI 기반 위협 탐지
- DevSecOps
- 데이터 유출 방지

HALO는 IAM 최소 권한 적용, 데이터 접근통제, 암호화 정책, 이상행위 탐지, API 보안 등을 통해 내부 정보 유출을 방지합니다.

<b>운영 안정성</b>
<br>특징</br>
- 자동 대응
- AI 기반 분석
- 장애 예측
- 이벤트 자동 분류
- Incident 자동 처리
운영 인력이 적어도 안정적인 운영이 가능합니다.

<b>규제 대응</b>
<br>지원</br>
- ISMS
- ISO27001
- 금융권 컴플라이언스
- 개인정보보호
가장 큰 특징은 vCISO 서비스를 함께 제공한다는 점입니다.

## SCENARIO 1. 멀티 클라우드 환경 (AWS + Azure)
여러 퍼블릭 클라우드를 동시에 사용할 때 가장 큰 보안 위협은 '가시성 분절(보안 사각지대)'과 '일관되지 않은 보안 정책'입니다.
<br>AWS와 Azure의 보안 설정 오류를 한곳에서 통제할 수 있어야 합니다.</br>

### [추천] 메가존클라우드 HALO & 베스핀글로벌 헬프나우
메가존클라우드 HALO (최적): 멀티 클라우드 환경에서 최고의 역량을 발휘합니다.
<br>AWS와 Azure의 네이티브 로그를 단일 대시보드로 통합(Single Pane of Glass)하고,</br>
글로벌 가상 보안 아키텍처인 지스케일러(Zscaler)와 연동해 사용자가 AWS에 접속하든 Azure에 접속하든 동일한 제로 트러스트 보안 정책을 적용합니다.

베스핀글로벌 헬프나우 (차선 - 가성비): 자체 멀티 클라우드 관리 플랫폼(CMP)인 OpsNow의 기술력이 녹아 있어,
<br>AWS와 Azure의 자원 설정 오류(CSPM)를 모니터링하는 가성비가 매우 훌륭합니다. 멀티 클라우드를 처음 도입해 비용과 복잡성이 고민인 조직에 좋습니다.</br>

삼성SDS / SK쉴더스: 대규모 멀티 클라우드 관제가 가능하지만,
<br>각 클라우드 간의 유기적인 연동보다는 개별 클라우드 인프라를 각각 관제·구축하는 성격이 강해 비용 부담이 커질 수 있습니다.</br>

## SCENARIO 2. 금융 전용망 환경 (금융보안원 규제 가이드라인 준수)
금융권 및 핀테크 환경은 망분리 규제(물리적/논리적), 전자금융감독규정 준수, CSAP 인증 등 대한민국에서 가장 까다로운 컴플라이언스를 만족해야 합니다.
<br>솔루션의 기능보다 '인증을 통과할 수 있는가'가 핵심입니다.</br>

### [추천] SK쉴더스 클라우드 보안 & 삼성SDS 클라우드 보안
SK쉴더스 (최적): 금융 전용망 아키텍처 설계와 금융보안원 심사 대응 분야에서 국내 독보적인 레퍼런스를 보유하고 있습니다.
<br>금융권의 핵심인 오픈뱅킹 API 보안, 망분리 예외 처리 요건(데이터 송수신 통제), 침해사고 대응(Top-CERT) 체계 구축까지 규제 통과를 위한 종합 컨설팅과 아키텍처링을 완벽히 지원합니다.</br>

삼성SDS (우수): 금융권 수준의 초고강도 보안이 적용된 자체 금융 전용 클라우드(SCP) 운영 노하우가 있습니다.
<br>금융 인프라에서 필수적인 강력한 계정 관리(SingleID), 암호화 키 관리(KMS), 국정원 인증 암호 모듈 적용 등 엔터프라이즈 레벨의 규제 준수 솔루션을 커스텀 패키징하여 제공합니다.</br>

메가존 / 베스핀글로벌: 퍼블릭 클라우드 기반의 보안에 특화되어 있어,
<br>엄격한 물리적 망분리나 폐쇄적인 금융 전용 인프라 환경의 특수 규제 요건을 100% 충족하기에는 컨설팅 깊이에서 아쉬움이 있을 수 있습니다.</br>

## SCENARIO 3. 온프레미스 하이브리드 환경 (Legacy + Cloud)
기존 사내 데이터 센터(Legacy 서버)를 유지하면서 일부 워크로드를 클라우드로 이전한 형태입니다.
<br>온프레미스의 하드웨어 장비(방화벽, IPS)와 클라우드의 가상 보안 솔루션이 유기적으로 연동되어 하나의 보안 경계를 형성해야 합니다.</br>

### [추천] 삼성SDS 클라우드 보안 & SK쉴더스 클라우드 보안
삼성SDS (최적): 수십 년간 대기업 계열사의 레거시 인프라를 운영해 온 하이브리드 전문가입니다.
<br>온프레미스 ERP/LOB 시스템과 퍼블릭 클라우드 간의 전용선(Direct Connect/Express Route) 구간 보안, 하이브리드 환경 전체를 아우르는 동적 보안 관제(Auto-scaling 연동) 역량이 매우 뛰어납니다. 내부 시스템 유출 방지를 위한 Managed CASB도 하이브리드 환경에서 강력하게 작용합니다.</br>

SK쉴더스 (우수): 전통적인 물리 보안 및 네트워크 온프레미스 관제(ADT캡스 기반) 인프라와 클라우드 관제 시스템을 결합한 하이브리드 소크(SIEM/SOAR) 운영에 강합니다.
<br>현장 장비 제어와 클라우드 제어가 동시에 필요한 제조·물류 기반 하이브리드 기업에 최적입니다.</br>

메가존 / 베스핀글로벌: 태생이 클라우드 네이티브(Cloud-Native) MSP이기 때문에,
<br>온프레미스 자체 데이터 센터의 하드웨어 보안 장비 교체, 물리적 망 구성, 전통적인 레거시 보안 아키텍처에 대한 제어 역량은 상대적으로 약할 수 있습니다.</br>

  <br>: 삼성SDS (or 기존 물리/전통 보안 장비까지 통합 관제하고 싶다면 SK쉴더스)</br>

# 🔒 ISMS-P / ISO 27001 대응: Shadow AI 및 소스코드 유출 방지 솔루션 비교

본 문서는 **ISMS-P** 및 **ISO 27001** 인증 유지 조건을 충족하면서, 임직원의 비인가 AI 서비스(ChatGPT, DeepL 등) 및 개인 클라우드 스토리지(Dropbox 등)를 통한 **사내 기밀 소스코드 유출을 차단**하기 위한 최적의 보안 서비스를 비교 분석한 결과 보고서입니다.

---

## 📋 컴플라이언스 만족을 위한 필수 기술 요건

깃허브(GitHub) 등 개발 환경에서 소스코드 유출을 원천 차단하고 대외 인증 심사를 통과하기 위해서는 아래 3가지 핵심 기능이 반드시 구현되어야 합니다.

1. **실시간 인라인 통제 (Inline Control):** API 방식의 사후 감시가 아닌, 사용자가 전송/업로드 버튼을 누르는 즉시 실시간으로 패킷을 분석하여 차단해야 합니다.
2. **섀도우 IT 식별 (Shadow IT Discovery):** 허가되지 않은 신종 AI 서비스 및 웹 스토리지 URL을 자동 분류하고 접근 권한을 제어할 수 있어야 합니다.
3. **감사 추적 및 증적 확보 (Audit Trail):** "누가, 언제, 어떤 소스코드(데이터)"를 유출하려 했는지 실시간 모니터링 로그가 남아야 심사 시 통제성 항목을 만족합니다.

* **컨텍스트 기반 세부 행위 제어:** 단순히 ChatGPT나 DeepL 접속을 전면 차단하는 것이 아니라, *"업무용 조회는 허용하되 프롬프트 창에 소스코드 복사-붙여넣기 및 파일 업로드는 차단"*하는 정밀 제어가 가능합니다.
* **소스코드 특화 DLP (데이터 유출 방지):** 텍스트 및 파일 내부의 소스코드 구조(정규식, 키워드, 특정 확장자 패턴)를 실시간 인식합니다. 개발자가 파일 확장자를 `.txt` 등으로 위변조하여 개인 드롭박스에 업로드하더라도 내부 코드를 탐지해 차단합니다.

### 🔹 컴플라이언스 (ISMS-P / ISO 27001) 강점
* HALO 보안 프레임워크를 통해 비인가 AI 서비스 사용 시도 현황을 실시간 자산화하고 감사 로그를 통합 수집하므로, 심사원에게 즉각적인 증적 제출이 가능합니다.

## 🏢 선정 솔루션 2: 삼성SDS Managed CASB
> **엔터프라이즈급 관리형 CASB 중심의 클라우드 위협 통제**

### 🔹 핵심 통제 기능
* **자동 Shadow IT Discovery:** 사내에서 외부로 나가는 모든 웹 트래픽을 정밀 분석하여, 보안 팀이 파악하지 못한 신종 번역/AI 사이트 및 웹 스토리지를 자동으로 식별하고 대시보드에 격리·차단 정책을 즉시 매핑합니다.

### 🔹 컴플라이언스 (ISMS-P / ISO 27001) 강점
* 국내 최대 규모의 보안 관제 인프라를 바탕으로, 대기업 감사 기준의 타이트한 통제 프로세스와 암호화 키 관리(KMS), 상세 유출 시도 리포트를 기본 제공합니다.

## 권장 방향

이 요구사항(컴플라이언스 + Shadow AI 차단 + 소스코드 유출 방지)에는 보통 **2개 레이어를 함께 구성**합니다.

### 1. AI-aware DLP (필수)
ChatGPT/DeepL 같은 웹 기반 AI 도구로 텍스트·코드를 붙여넣는 행위 자체를 탐지·차단

- **소만사 WebKeeper AI DLP** — 1금융권 5곳 운용 중이라는 레퍼런스가 가장 구체적. 망분리 환경(N2SF) 최적화도 명시돼 있어 금융 전용망 시나리오와도 부합
- **파수 AI-R DLP** — 유사한 포지션이며 국내 데이터보안 전문기업으로 업력이 깊음

### 2. 문서 DRM/암호화 (보완)
소스코드 파일 자체가 드롭박스 등으로 반출되더라도 열람 불가능하게 만드는 2차 방어

- **마크애니 SafePC** — 생성 시점 자동 암호화 + 외부 반출 시 열람 차단을 동시에 제공해 코드 유출 방지에 적합
- **지란지교 오피스키퍼** — 조달청 등록, CC/GS 인증으로 공공·일반기업 어디든 컴플라이언스 입증이 용이

> **참고**: 현재 멀티클라우드(AWS/Azure) 환경에 적용할 **인프라 보안(메가존 HALO 등)**과 이번 **Shadow AI/DLP 솔루션**은 서로 대체재가 아니라 **보완재**입니다. 인프라 단의 클라우드 보안과 엔드포인트 단의 정보유출방지(DLP)는 별도 트랙으로 RFP를 진행하는 것이 일반적입니다.

# 🔐 IdP(Okta/Entra ID) 연동 환경에서의 SaaS 보안 솔루션 비교 분석

본 보고서는 사내에서 이미 도입하여 운용 중인 **Okta** 및 **Azure AD (Entra ID)** 계정 관리(IdP) 인프라를 기반으로, **메가존클라우드 HALO**와 **삼성SDS Managed CASB** 도입 시의 **[1] 연동성**, **[2] 운영 공수(Effort)**, **[3] 비용 확장성(Cost Scalability)**을 비교 분석한 결과입니다.

---

## 🏗️ 기존 IdP (Okta / Entra ID) 연동 아키텍처 비교

이미 구축된 계정 관리 체계를 활용해 중복 투자를 방지하고, **'조건부 액세스(Conditional Access)'** 및 **'싱글사인온(SSO)'** 정책을 결합하는 방식에서 두 솔루션은 구조적 차이를 보입니다.

### 1. 메가존클라우드 HALO (Zscaler SSE 기반)
* **연동 메커니즘:** Zscaler 인프라가 Okta/Entra ID와 SAML 2.0 및 SCIM 프로토콜로 완벽하게 연동됩니다. 사용자 계정 및 그룹 정보가 실시간 동기화됩니다.
* **통제 방식:** Okta에서 인증을 마친 사용자의 단말 트래픽이 HALO를 거칠 때, IdP의 그룹 정보(예: 개발팀, 인사팀)를 그대로 이어받아 차등 정책을 적용합니다. (예: 개발팀은 ChatGPT 조회 가능하나 소스코드 입력 불가, 일반 부서는 ChatGPT 접속 완전 차단)

### 2. 삼성SDS Managed CASB
* **연동 메커니즘:** Entra ID 및 Okta의 API 및 로그 연동을 통해 SaaS 접근 이력을 모니터링하고 가시성을 확보합니다.
* **통제 방식:** IdP 자체의 '조건부 액세스' 기능과 CASB의 프록시/API 통제 기능을 결합하여 다중 인증(MFA) 및 비인가 기기 접속 제한을 구현합니다. 삼성의 통합 계정 관리(SingleID) 플랫폼 노하우가 결합되어 국내 환경에 최적화된 연동 리포트를 제공합니다.

## 🔍 핵심 비교 분석 딥다이브

### 💡 1. 운영 공수(Operation Effort) 관점
* **메가존클라우드 HALO 승:** HALO가 기반으로 하는 Zscaler는 글로벌 섀도우 IT 및 AI 서비스 데이터베이스(DB)를 알아서 매일 업데이트합니다. 새로운 AI 번역기나 클라우드가 등장해도 보안 담당자가 일일이 URL을 등록할 필요가 없어 **정책 유지보수 공수가 거의 제로**에 가깝습니다.
* **삼성SDS Managed CASB:** 삼성의 Managed 서비스는 대시보드를 직접 관리하기 어려운 기업에게 관제 인력을 제공하므로 편리하지만, 사내 정책 변경이나 예외 처리를 요청하고 조율하는 과정에서 프로세스적 공수가 발생할 수 있습니다.

### 💰 2. 비용 확장성(Cost Scalability) 관점
* **메가존클라우드 HALO 승:** 글로벌 SaaS 특성상 **사용자당 구독 라이선스(Per-User)** 체계가 명확합니다. 임직원 수가 늘어나거나, 재택근무자가 급증하거나, AWS/Azure 인프라가 대폭 확장되어도 네트워크 장비 증설 없이 라이선스만 추가하면 되므로 예측 가능성이 매우 높습니다.
* **삼성SDS Managed CASB:** 엔터프라이즈 맞춤형 구축 및 관제(Managed) 성격이 결합되어 있어, 초기 연동 구축 비용 및 관제 범위 확장(연동할 SaaS 개수 증가 등)에 따른 추가 비용 구조를 면밀히 따져야 하므로 대규모 확장 시 단가 부담이 발생할 수 있습니다.

# Okta + Microsoft Entra ID 기반 환경에서 SaaS 보안 솔루션 비교
## (메가존 HALO vs 삼성 SDS CASB)

---

# 1. 검토 배경

현재 사내 인증 체계는 다음과 같이 운영되고 있다.

- Okta (Identity Provider)
- Microsoft Entra ID(Azure AD)
- AWS
- Azure
- SaaS 서비스

또한 향후 도입 예정인 SaaS 보안 솔루션은 다음 조건을 만족해야 한다.

- 기존 Okta와 연동 가능
- Entra ID와 연동 가능
- ISMS-P 준수
- ISO/IEC 27001 준수
- Shadow AI 통제
- SaaS 접근 제어
- 운영 공수 최소화
- 향후 SaaS 증가 시 확장성 확보

```text

            ┌───────────┴───────────┐

      Microsoft Entra ID         SaaS

            │

            ▼

     AWS / Azure / M365

            │

            ▼

      신규 SaaS 보안 솔루션
```

# 4. 메가존 HALO

## 특징

HALO는 클라우드 보안 운영 플랫폼(CNAPP, AI-SOC, CSPM 중심)으로 설계되어 있으며,

- AWS
- Azure
- GCP

멀티클라우드 운영 자동화에 강점을 가진다.

또한

- AI SOC
- AI 기반 자동 대응
- 통합 모니터링
- Compliance 지원

기능을 제공한다.

---

## Okta 연동

가능

지원 방식

- SAML
- OIDC
- SCIM
- 기존 IdP 연계

Okta를 그대로 유지하면서 연동 가능하다.

---

## Entra ID 연동

가능

Azure AD와의 인증 체계를 그대로 사용할 수 있다.

---

## 운영 공수

★★★★★

장점

- 운영 자동화
- AI 기반 분석
- 자동 플레이북
- AI SOC

운영 인력이 적은 기업에서도 관리 부담이 낮다.

---

## 확장성

★★★★★

새로운 SaaS가 증가하여도

HALO에서 정책을 통합 관리할 수 있어 운영 복잡도가 비교적 낮다.

---

## 비용

초기 구축 비용은 존재하지만,

장기적으로는 운영 자동화 효과로 운영 비용(TCO)을 절감할 가능성이 높다.

---

# 5. 삼성 SDS CASB

## 특징

삼성 SDS CASB는

Cloud Access Security Broker 기반으로

- SaaS 접근 통제
- Shadow AI 통제
- DLP
- SWG
- Zero Trust

중심의 보안 솔루션이다.

---

## Okta 연동

가능

Okta 기반

- SAML
- OIDC
- SCIM

연동 가능

기존 인증 체계를 유지할 수 있다.

---

## Entra ID 연동

매우 우수

특히

Microsoft 365

Azure

OneDrive

Teams

연동이 뛰어나다.

---

## 운영 공수

★★★☆☆

CASB 정책

- 애플리케이션별 정책
- 사용자 그룹별 정책
- 예외 정책
- DLP 정책

등을 지속적으로 관리해야 하므로

운영 공수가 상대적으로 높다.

---

## 확장성

★★★★☆

SaaS가 증가하여도

CASB 정책을 계속 추가하면 되지만,

서비스가 많아질수록 정책 관리 복잡도가 증가한다.

---

## 비용

초기 구축 비용과 정책 설계 비용이 비교적 높은 편이며,

운영 인력 투입도 지속적으로 필요하다.

# 10. 최종 결론

현재와 같이

- Okta
- Microsoft Entra ID
- AWS
- Azure

환경을 이미 구축한 기업이라면

인증 연동 측면에서는 두 솔루션 모두 큰 차이가 없다.

그러나 평가 기준이

- 운영 공수
- 운영 자동화
- 장기 운영 비용
- SaaS 증가 시 확장성

이라면

**메가존 HALO가 상대적으로 유리하다.**

반대로

- Shadow AI 통제
- CASB
- DLP
- SaaS 접근 제어
- 데이터 유출 방지

를 가장 중요한 목표로 한다면

**삼성 SDS CASB가 보다 적합하다.**
